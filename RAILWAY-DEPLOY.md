# Деплой на Railway - Инструкция

## Быстрый старт

### Шаг 1: Подготовка GitHub репозитория

1. Создайте репозиторий на GitHub.com
2. Загрузите все файлы проекта (кроме node_modules)
3. **Важно:** Файл `standalone.html` должен быть в корне репозитория

### Шаг 2: Деплой на Railway

1. Зайдите на [railway.app](https://railway.app/)
2. Нажмите **"New Project"**
3. Выберите **"Deploy from GitHub repo"**
4. Выберите ваш репозиторий
5. Railway автоматически:
   - Определит Node.js проект
   - Установит зависимости
   - Запустит сервер

### Шаг 3: Получите URL

После деплоя Railway предоставит URL:
- `https://your-project-name.up.railway.app`

Сайт будет доступен по адресу:
- `https://your-project-name.up.railway.app/standalone.html`

---

## Что нужно загрузить в GitHub

✅ **Обязательные файлы:**
- `standalone.html` - главный файл сайта
- `package.json` - зависимости
- `Procfile` - команда запуска
- `railway.json` - конфигурация Railway
- `.gitignore` - игнорируемые файлы

❌ **НЕ загружайте:**
- `node_modules/` - создастся автоматически
- `.vscode/` - настройки редактора
- Логи и временные файлы

---

## Структура для Railway

```
Landing-project/
├── standalone.html      ← Главный файл
├── package.json        ← Зависимости
├── Procfile            ← Команда запуска
├── railway.json        ← Конфигурация
└── .gitignore          ← Игнорируемые файлы
```

---

## Проверка деплоя

После деплоя проверьте:
1. Логи в Railway Dashboard - не должно быть ошибок
2. URL работает - откройте в браузере
3. Сайт загружается корректно

---

## Обновление сайта

После каждого изменения:
1. Загрузите изменения в GitHub
2. Railway автоматически перезадеплоит проект
3. Новые изменения появятся через 1-2 минуты

---

## Настройка домена

1. В Railway Dashboard → Settings → Domains
2. Добавьте свой домен или используйте Railway домен
3. Настройте DNS записи (если используете свой домен)

---

## Проблемы и решения

### Сайт не открывается
- Проверьте логи в Railway Dashboard
- Убедитесь, что `standalone.html` в корне проекта
- Проверьте, что порт использует `$PORT`

### Ошибка при сборке
- Проверьте, что `serve` в dependencies
- Убедитесь, что все файлы загружены в GitHub

### 404 ошибка
- Откройте `/standalone.html` явно
- Или настройте редирект с корня на standalone.html
